## 🔐 Required GitHub Secrets

To enable automatic deployments, configure these secrets in your GitHub repository:

### Step 1: Get Vercel Credentials
1. Go to [Vercel Account Settings](https://vercel.com/account/tokens)
2. Create a new token: "Tokens" → "Create" → Name it (e.g., "GitHub Actions")
3. Copy the token value

### Step 2: Get Vercel Project IDs
1. Visit [Vercel Dashboard](https://vercel.com/dashboard)
2. Select "qawafel-admin" project
3. Go to "Settings" → "General"
4. Copy:
   - **Project ID** - shown in Settings
   - **Team ID/Org ID** - shown in Settings

### Step 3: Add GitHub Secrets
Go to your GitHub repository:
1. **Settings** → **Secrets and variables** → **Actions**
2. Click **"New repository secret"** and add:

```
Name: VERCEL_TOKEN
Value: <paste token from Step 1>

Name: VERCEL_ORG_ID
Value: <paste Team/Org ID from Step 2>

Name: VERCEL_PROJECT_ID
Value: <paste Project ID from Step 2>

Name: NEXT_PUBLIC_API_URL
Value: https://qawafel-api-production.up.railway.app
```

## 🚀 Deployment Workflow

### Automatic Deployment (On Push to Main)
```
1. Developer pushes code to `main` branch
   ↓
2. GitHub Actions workflow triggers automatically
   ↓
3. Build job runs:
   - Checkout code
   - Install dependencies
   - Run ESLint checks
   - Build Next.js production bundle
   ↓
4. If build succeeds → Deploy job runs:
   - Deploy to Vercel (production environment)
   - Built artifact uploaded to CDN
   ↓
5. Vercel assigns unique deployment URL
   ↓
6. Production live at vercel.com dashboard
```

### Pull Request (CI Only)
- Build and lint checks run on every PR
- Deployment does NOT occur for pull requests
- Vercel preview deployment can be enabled separately in Vercel dashboard

## 🔄 Backend API Integration

The frontend calls your Railway-hosted backend at:
```
https://qawafel-api-production.up.railway.app
```

**Required backend endpoints** (same as development):
- `POST /auth/login`
- `POST /auth/refresh`
- `GET /offers`
- `GET /agencies`
- `GET /destinations`
- `GET /hotels`
- `GET /packages`
- `GET /admin/users` (admin only)
- `GET /admin/audit-logs` (admin only)
- ...and other CRUD operations

**CORS Setup Required:**
Backend must allow requests from:
- `http://localhost:3000` (development)
- `https://qawafel-admin-*.vercel.app` (production)

## 🐛 Troubleshooting

### Build Fails
1. Check GitHub Actions logs: Repository → Actions tab
2. Common issues:
   - TypeScript errors (run `npm run build` locally)
   - Missing environment variables (verify GitHub secrets)
   - Dependency conflicts (clear node_modules: `rm -rf node_modules && npm install`)

### Deployment Fails
1. Verify Vercel secrets are correct: copy-paste from Settings, no extra spaces
2. Check Vercel dashboard build logs
3. Ensure backend API URL is accessible and CORS headers are set

### App Works But Can't Call API
1. Verify `NEXT_PUBLIC_API_URL` in deployed app (check browser console)
2. Check browser DevTools → Network tab for failed API requests
3. Confirm backend CORS headers include Vercel domain
4. Test backend directly: `curl https://qawafel-api-production.up.railway.app/health`
